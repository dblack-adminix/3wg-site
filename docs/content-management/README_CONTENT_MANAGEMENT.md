# 🎛️ Система управления контентом главной страницы

## ✅ Статус: Полностью реализовано

Система позволяет включать/отключать и **перемещать** 11 блоков на главной странице через админку.

## 🚀 Быстрый старт

### 1. Откройте админку
```
http://localhost:8080/admin
```

### 2. Войдите как админ
- **Email**: `admin@3wg.ru`
- **Пароль**: `admin123`

### 3. Перейдите на вкладку "Контент сайта"

### 4. Управляйте блоками
- **Включить/отключить**: Нажмите на иконку глаза
- **Переместить**: Перетащите блок за иконку ⋮⋮
- **Сохранить**: Нажмите "Сохранить" для применения изменений
- **Обновить**: Обновите главную страницу для просмотра результата

## 🎯 Новое: Drag & Drop!

Теперь можно **менять порядок блоков** простым перетаскиванием!

### Как переместить блок:
1. Наведите на иконку **⋮⋮** слева от блока
2. Зажмите и перетащите вверх/вниз
3. Отпустите в нужном месте
4. Нажмите "Сохранить"

### Пример:
Хотите поставить "Тарифы" выше "VPN секции"? Просто перетащите блок "Тарифы" на позицию #3!

**Важно**: Hero секция всегда остается первой (фиксированная).

## 🔌 API

### Получить настройки (публичный доступ)
```bash
GET http://localhost:3000/api/v1/settings/homepage
```

Ответ:
```json
{
  "hero_section": true,
  "keenetic_section": true,
  "vpn_section": true,
  "services_section": true,
  "pricing_section": true,
  "hardware_section": true,
  "infrastructure_section": true,
  "faq_section": true,
  "articles_section": true,
  "telegram_section": true,
  "status_widget": true
}
```

### Обновить настройки (требуется токен админа)
```bash
PUT http://localhost:3000/api/v1/admin/settings/homepage
Authorization: Bearer <token>
Content-Type: application/json

{
  "hero_section": false,
  "keenetic_section": true,
  ...
}
```

## 🎯 Как это работает

1. **Backend** хранит настройки в таблице `site_settings` в PostgreSQL
2. **API** предоставляет публичный эндпоинт для получения настроек
3. **Frontend** загружает настройки при загрузке главной страницы
4. **Условный рендеринг** показывает/скрывает блоки на основе настроек
5. **Админка** позволяет изменять настройки через UI

## ✨ Особенности

- ✅ **Включение/отключение блоков**: Простое управление видимостью
- ✅ **Drag & Drop**: Перемещение блоков перетаскиванием
- ✅ **Фиксированный Hero**: Hero секция всегда первая
- ✅ **Реактивность**: Изменения применяются сразу после сохранения
- ✅ **Дефолтные значения**: Если настройки не найдены, все блоки включены
- ✅ **Публичный доступ**: Настройки доступны без авторизации
- ✅ **Админ контроль**: Изменение только через админку с JWT токеном
- ✅ **Визуальная обратная связь**: Зеленые/красные индикаторы статуса
- ✅ **Отмена изменений**: Кнопка "Отменить" для сброса несохраненных изменений
- ✅ **Нумерация блоков**: Каждый блок показывает свою позицию (#1, #2, #3...)
- ✅ **Клавиатурная навигация**: Полная поддержка управления с клавиатуры

### Тест 2: Включить блок через API
```powershell
# Получить токен
$login = @{ email = "admin@3wg.ru"; password = "admin123" } | ConvertTo-Json
$response = Invoke-RestMethod -Uri "http://localhost:3000/api/v1/auth/login" -Method POST -Body $login -ContentType "application/json"
$token = $response.token

# Включить блок FAQ
$settings = @{ faq_section = $true; ... } | ConvertTo-Json
$headers = @{ "Authorization" = "Bearer $token" }
Invoke-RestMethod -Uri "http://localhost:3000/api/v1/admin/settings/homepage" -Method PUT -Body $settings -Headers $headers -ContentType "application/json"
```

### Тест 3: Проверить дефолтные значения
1. Удалите запись из таблицы `site_settings`
2. Откройте главную страницу
3. **Ожидаемый результат**: Все блоки включены

## 🐛 Troubleshooting

### Проблема: Блоки не исчезают
**Решение**: Обновите страницу (F5) или очистите кеш (Ctrl+Shift+R)

### Проблема: Ошибка при сохранении
**Решение**: Проверьте, что бэкенд запущен и токен валиден

### Проблема: Все блоки отключены
**Решение**: Запустите `.\enable-all-blocks.ps1`
