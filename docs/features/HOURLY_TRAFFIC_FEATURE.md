## Как это работает

1. **Сбор данных**:
   - При первом запуске воркер заполняет последние 24 часа (backfill)
   - Backfill распределяет текущий трафик равномерно по часам (для визуализации)
   - Каждый час воркер делает снимок текущего трафика всех пиров
   - Сохраняет кумулятивные значения (total_receive, total_sent)
   - Рассчитывает дельту (разницу с предыдущим снимком) - это трафик за час

2. **Хранение**:
   - Данные хранятся в таблице `peer_traffic_hourly`
   - Каждая запись = 1 час для 1 пира
   - Индексы обеспечивают быструю выборку

3. **Отображение**:
   - Frontend запрашивает последние 24 часа при открытии деталей пира
   - Recharts рисует линейный график с почасовыми точками
   - Показывается трафик получен/отправлен за каждый час
   - Внизу - суммарная статистика за 24 часа

## Важные моменты

1. **Первый запуск (Backfill)**:
   - При первом запуске воркер заполняет последние 24 часа
   - Backfill распределяет текущий трафик равномерно (линейная интерполяция)
   - Это не точные данные, но позволяет показать график сразу
   - Со следующего часа начинают собираться реальные данные

2. **Реальные данные**:
   - После backfill каждый час сохраняется реальный снимок
   - Дельта = реальный трафик за последний час
   - Через 24 часа весь график будет состоять из реальных данных

3. **Частота обновления**:
   - Воркер запускается каждый час (в начале часа)
   - Frontend загружает данные при открытии модалки
   - Данные не обновляются автоматически (нужно закрыть/открыть модалку)

4. **Производительность**:
   - Почасовые снимки легковесные (только дельта)
   - Индексы обеспечивают быструю выборку
   - Запрос 24 часов = 24 записи (очень быстро)

5. **Очистка старых данных**:
   - Пока не реализована автоматическая очистка
   - Рекомендуется хранить 7-30 дней почасовых данных
   - Можно добавить cron job для удаления старых записей
