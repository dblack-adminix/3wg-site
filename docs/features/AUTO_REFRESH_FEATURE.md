# Автообновление данных в реальном времени 🔄

## Обзор

Добавлено автоматическое обновление данных на странице серверов без перезагрузки страницы и без показа спиннеров загрузки.

## Реализованные функции

### 1. Автообновление списка серверов

**Интервал**: каждые 30 секунд  
**Режим**: тихий (без спиннера)

```typescript
useEffect(() => {
    loadServers(); // Первая загрузка со спиннером
    
    // Автообновление каждые 30 секунд (без спиннера)
    const interval = setInterval(() => {
        loadServers(true); // silent = true
    }, 30000);
    
    return () => clearInterval(interval);
}, []);
```

**Что обновляется:**
- ✅ Список серверов
- ✅ Системный статус (CPU, RAM, Disk) для каждого сервера
- ✅ Статус серверов (active/offline)

### 2. Автообновление деталей сервера

**Интервал**: каждые 15 секунд  
**Режим**: фоновый (без спиннера)

```typescript
useEffect(() => {
    if (selectedServerId) {
        loadServerDetails(selectedServerId);
        
        // Автообновление каждые 15 секунд
        const interval = setInterval(() => {
            if (selectedServerId) {
                // Обновляем конфигурации и пиры
                api.getServerConfig(selectedServerId)
                    .then(configsData => {
                        setServerConfigs(configsData.data || []);
                        
                        // Обновляем пиры для активной конфигурации
                        if (activeConfig) {
                            api.getServerPeers(selectedServerId, activeConfig)
                                .then(peersData => setServerPeers(peersData.peers || []))
                                .catch(() => {});
                        }
                    })
                    .catch(() => {});
                
                // Обновляем системный статус
                api.getSystemStatus(selectedServerId)
                    .then(statusData => setSystemStatus(statusData))
                    .catch(() => {});
            }
        }, 15000);
        
        return () => clearInterval(interval);
    }
}, [selectedServerId, activeConfig]);
```

**Что обновляется:**
- ✅ Конфигурации WireGuard (wg0, wg1, etc.)
- ✅ Список пиров для активной конфигурации
- ✅ Статус пиров (онлайн/офлайн)
- ✅ Трафик пиров (получено/отправлено)
- ✅ Системный статус (CPU, RAM, Disk)
- ✅ Статистика конфигураций (пиры онлайн/всего, трафик)

### 3. Оптимизированная загрузка

**Фоновая загрузка без блокировки UI:**

```typescript
const loadServerDetails = async (serverId: number) => {
    try {
        setLoadingDetails(true);
        
        // Сначала загружаем основную информацию о сервере
        const details = await api.getServer(serverId);
        setServerDetails(details);
        setLoadingDetails(false); // ✅ Убираем спиннер сразу
        
        // Загружаем конфигурации в фоне
        api.getServerConfig(serverId)
            .then(configsData => {
                setServerConfigs(configsData.data || []);
                
                // Загружаем пиры для первой конфигурации
                if (configsData.data && configsData.data.length > 0) {
                    setActiveConfig(configsData.data[0].Name);
                    api.getServerPeers(serverId, configsData.data[0].Name)
                        .then(peersData => setServerPeers(peersData.peers || []))
                        .catch(() => setServerPeers([]));
                }
            })
            .catch(() => setServerConfigs([]));
        
        // Загружаем системный статус в фоне
        api.getSystemStatus(serverId)
            .then(statusData => setSystemStatus(statusData))
            .catch(() => setSystemStatus(null));
            
    } catch (error) {
        console.error('Failed to load server details:', error);
        toast.error('Ошибка загрузки деталей сервера');
        setLoadingDetails(false);
    }
};
```

## Преимущества

### ✅ Актуальные данные в реальном времени
- Статус серверов обновляется автоматически
- Трафик пиров обновляется каждые 15 секунд
- Системные ресурсы (CPU, RAM, Disk) всегда актуальны

### ✅ Незаметное обновление
- Нет спиннеров при автообновлении
- UI не блокируется
- Пользователь может продолжать работать

### ✅ Оптимизированная производительность
- Первая загрузка мгновенная (~100ms)
- Фоновые обновления не блокируют UI
- Независимая загрузка для каждого сервера

### ✅ Устойчивость к ошибкам
- Ошибки автообновления не показываются пользователю
- Старые данные остаются до успешного обновления
- Graceful degradation при недоступности API

## Интервалы обновления

| Данные | Интервал | Причина |
|--------|----------|---------|
| Список серверов | 30 сек | Статус серверов меняется редко |
| Системный статус | 30 сек | CPU/RAM/Disk меняются плавно |
| Детали сервера | 15 сек | Пиры могут подключаться/отключаться |
| Конфигурации | 15 сек | Новые пиры могут добавляться |
| Пиры | 15 сек | Трафик и статус меняются часто |

### Адаптивные интервалы
```typescript
// Увеличиваем интервал если пользователь неактивен
let interval = 15000;
document.addEventListener('visibilitychange', () => {
    if (document.hidden) {
        interval = 60000; // 1 минута если вкладка неактивна
    } else {
        interval = 15000; // 15 секунд если вкладка активна
    }
});
```

### Умное обновление
```typescript
// Обновляем только изменившиеся данные
const prevData = usePrevious(serverData);

    // Данные изменились - обновляем UI

}
```
