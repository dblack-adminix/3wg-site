# Исправление z-index для модальных окон пиров

## Проблема
Модальные окна (Dialog с деталями пира и AlertDialog удаления пира) открывались, но находились ЗА основным контентом из-за неправильного z-index. Модалка была видна только когда пользователь возвращался к списку серверов.

## Решение
Добавлен `z-[200]` для обоих диалогов:

### 1. Dialog с деталями пира
```tsx
<Dialog open={isPeerDetailsDialogOpen} onOpenChange={setIsPeerDetailsDialogOpen}>
  <DialogContent className="max-w-4xl max-h-[90vh] overflow-y-auto bg-[#0a0a0a] border-border z-[200]">
```

### 2. AlertDialog удаления пира
```tsx
<AlertDialog open={isDeletePeerDialogOpen} onOpenChange={setIsDeletePeerDialogOpen}>
  <AlertDialogContent className="bg-[#0a0a0a] border-border z-[200]">
```

## Дополнительные изменения
- Убраны `console.log()` для отладки в обработчиках кликов
- Используется `z-[200]` вместо `z-[100]` для избежания конфликтов с другими компонентами (OrderForm, MobileDashboard)
