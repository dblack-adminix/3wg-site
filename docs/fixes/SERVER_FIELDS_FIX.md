# Исправление отображения полей серверов

## Проблема

В карточках серверов не отображались:
1. **Протоколы** - показывало "N/A"
2. **Локация** - не было флага и местоположения

## Причина

У некоторых серверов в БД были пустые поля:
- `location` - NULL или пустая строка
- `country` - NULL или пустая строка
- `protocols` - NULL или пустой массив

## Решение

### 1. Обновлены данные в БД

```sql
-- Сервер #1
UPDATE servers 
SET location = 'Amsterdam', 
    country = 'NL', 
    protocols = '{wireguard}' 
WHERE id = 1;

-- Сервер #4
UPDATE servers 
SET location = 'Moscow', 
    country = 'RU', 
    protocols = '{wireguard,amneziawg}' 
WHERE id = 4;
```

### 2. Добавлены поля в форму создания/редактирования

**Файл:** `src/components/admin/ServersTab.tsx`

**Добавлены поля:**
- **Локация** - текстовое поле (например, "Amsterdam", "Moscow")
- **Страна (код)** - текстовое поле, 2 символа, uppercase (например, "NL", "RU", "US")
- **Протоколы** - чекбоксы для выбора:
  - WireGuard
  - AmneziaWG

**Изменения в formData:**
```typescript
const [formData, setFormData] = useState({
  name: '',
  location: '',        // ← Добавлено
  country: '',         // ← Добавлено
  protocols: ['wireguard'] as string[], // ← Добавлено
  wg_dashboard_url: '',
  wg_dashboard_key: '',
  max_users: 100,
});
```

**Изменения в handleOpenDialog:**
```typescript
if (server) {
  setFormData({
    name: server.name,
    location: server.location || '',
    country: server.country || '',
    protocols: server.protocols || ['wireguard'],
    wg_dashboard_url: server.wg_dashboard_url || '',
    wg_dashboard_key: server.wg_dashboard_key || '',
    max_users: server.max_users || 100,
  });
}
```

### 3. Добавлена валидация

Кнопка "Создать/Обновить" теперь требует:
- ✅ Название сервера

- ✅ Код страны

## Backend поддержка

Backend уже полностью поддерживает эти поля:

### Модель Server
```go
type Server struct {
    Location   string         `gorm:"not null" json:"location"`
    Country    string         `gorm:"not null" json:"country"`
    Protocols  pq.StringArray `gorm:"type:text[]" json:"protocols"`
    // ...
}
```

### CreateServer
- Принимает `location`, `country`, `protocols`
- Автоматически определяет локацию по IP если не указана
- Устанавливает дефолтные значения

### UpdateServer
- Обновляет все поля включая `location`, `country`, `protocols`

## Коды стран

Используются двухбуквенные коды ISO 3166-1 alpha-2:

| Код | Страна |
|-----|--------|
| NL  | Нидерланды |
| RU  | Россия |
| US  | США |
| DE  | Германия |
| FR  | Франция |
| GB  | Великобритания |
| JP  | Япония |
| SG  | Сингапур |

Полный список: https://en.wikipedia.org/wiki/ISO_3166-1_alpha-2

## Протоколы

Поддерживаемые протоколы:
- **wireguard** - стандартный WireGuard
- **amneziawg** - AmneziaWG (обфусцированный WireGuard)

Можно выбрать несколько протоколов одновременно.

## Как использовать

### Создание нового сервера

1. Откройте админку → "Серверы"
2. Нажмите "Добавить сервер"
3. Заполните поля:
   - **Название**: Amsterdam Server
   - **Локация**: Amsterdam
   - **Страна**: NL
   - **Протоколы**: ☑ WireGuard
   - **URL WGDashboard**: http://46.30.43.35:10086
   - **API ключ**: ваш ключ
   - **Максимум пиров**: 100
4. Нажмите "Создать"

### Редактирование существующего сервера

1. Откройте админку → "Серверы"
2. Найдите сервер
3. Нажмите кнопку "✏" (редактировать)
4. Обновите поля
5. Нажмите "Обновить"
