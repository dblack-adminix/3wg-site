## Как протестировать

### 1. Откройте админ-панель
- Перейдите на http://localhost:8080/admin
- Войдите как admin@3wg.ru / admin123

### 2. Откройте детали сервера
- Перейдите на вкладку "Серверы"
- Нажмите "Подробнее" на любом сервере с WGDashboard

### 3. Выберите конфигурацию
- Вверху страницы будут вкладки с конфигурациями (wg0, wg1, и т.д.)
- Выберите конфигурацию в которую хотите добавить пир

### 4. Откройте диалог добавления пира
- Нажмите кнопку "Добавить пир" (справа от заголовка "Пиры")
- Должен открыться диалог с формой

### 5. Заполните форму

**ВАЖНО**: Для добавления пира нужен валидный WireGuard публичный ключ!

#### Генерация ключей

Если у вас установлен WireGuard:
```bash
# Linux/Mac
wg genkey | tee privatekey | wg pubkey > publickey
cat publickey

# Windows (PowerShell с WireGuard)
wg genkey | wg pubkey
```

Или используйте онлайн генератор: https://www.wireguardconfig.com/

#### Пример заполнения формы:
- **Имя пира**: `my-laptop` или `test-device-01`
- **Публичный ключ**: Вставьте сгенерированный публичный ключ (44 символа base64)
  ```
  Пример: 8Jx9vkKt5Qy7wN2mP3rL4sT6uV8xZ0aB1cD2eF3gH4i=
  ```
- **Разрешенные IP**: `10.0.0.100/32` (или любой свободный IP из подсети конфигурации)

**Примечание**: Можно указать несколько IP через запятую, например: `10.0.0.100/32, 10.0.1.100/32`

### 6. Добавьте пир
- Нажмите кнопку "Добавить"
- Должно появиться уведомление "Пир добавлен"
- Список пиров должен обновиться автоматически

## Что происходит под капотом

1. Фронтенд отправляет POST запрос на `/api/v1/admin/servers/:id/wg/peers` с данными:
   ```json
   {
     "name": "test-peer-01",
     "public_key": "abcdefghijklmnopqrstuvwxyz0123456789ABCD=",
     "allowed_ips": ["10.0.0.100/32"],
     "config": "wg0"
   }
   ```

2. Бэкенд:
   - Получает сервер из БД
   - Создает клиент WGDashboard с указанной конфигурацией
   - Вызывает метод `AddPeer` который:
     - Получает текущую raw конфигурацию через `/api/getWireguardConfigurationRawFile`
     - Добавляет секцию `[Peer]` с данными пира
     - Обновляет конфигурацию через `/api/updateWireguardConfigurationRawFile`

3. После успешного добавления:
   - Фронтенд перезагружает список пиров
   - Новый пир появляется в списке

## Возможные проблемы

### Кнопка "Добавить" неактивна
- Убедитесь что все поля заполнены (имя, публичный ключ, разрешенные IP)
- Проверьте что выбрана конфигурация (activeConfig не пустой)

### Диалог не открывается
- Проверьте консоль браузера на ошибки (F12)
- Убедитесь что вы находитесь в режиме просмотра деталей сервера (selectedServerId установлен)

### Ошибка при добавлении
- Проверьте что публичный ключ валидный (base64, 44 символа)
- Проверьте что IP адрес в правильном формате (например, 10.0.0.100/32)
- Проверьте логи бэкенда: `docker-compose logs -f backend`
