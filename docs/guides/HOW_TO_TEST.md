# Как протестировать генератор ключей

## 🚀 Быстрый старт

### 1. Откройте браузер
```
http://localhost:8080/generator
```

### 2. Вас перенаправит на страницу логина
```
http://localhost:8080/login
```

### 3. Зарегистрируйтесь или войдите

**Вариант А: Создать нового пользователя**
1. На странице логина переключитесь на "Регистрация"
2. Введите email (например, `test@example.com`)
3. Введите пароль (например, `password123`)
4. Нажмите "Зарегистрироваться"

**Вариант Б: Использовать готового пользователя**
```powershell
# Создать через API
./create-test-user.ps1
```
Затем войдите:
- Email: `user@test.com`
- Password: `password123`

**Вариант В: Войти как админ**
- Email: `admin@3wg.ru`
- Password: `admin123`

### 4. После входа вас перенаправит на Generator

Теперь можете:
1. Выбрать протокол (WireGuard / AmneziaWG)
2. Выбрать сервер
3. Ввести имя ключа
4. Нажать "GENERATE CONFIG"
5. Скачать или скопировать конфиг

### 5. Проверить созданные ключи
```
http://localhost:8080/keys
```

## 🔐 Как работает авторизация

1. **Регистрация** → создается пользователь в БД
2. **Логин** → получаете JWT токен
3. **Токен** → сохраняется в localStorage
4. **Защищенные страницы** → требуют токен
5. **Logout** → токен удаляется

## 📱 Навигация

После входа доступны:
- `/dashboard` - Личный кабинет
- `/generator` - Генератор ключей ⭐
- `/keys` - Управление ключами
- `/account` - Настройки аккаунта
- `/deposit` - Пополнение баланса

## 🧪 Тестовые сценарии

### Сценарий 1: Новый пользователь
1. Откройте `/login`
2. Переключитесь на "Регистрация"
3. Зарегистрируйтесь
4. Перейдите на `/generator`
5. Создайте ключ

### Сценарий 2: Существующий пользователь
1. Откройте `/login`
2. Войдите (user@test.com / password123)
3. Перейдите на `/generator`
4. Создайте ключ

### Сценарий 3: Без авторизации
1. Откройте `/generator` напрямую
2. Вас перенаправит на `/login`
3. После входа вернет на `/generator`

## 🐛 Возможные проблемы

### "Unauthorized" при создании ключа
**Причина:** Токен истек или невалидный
**Решение:** Выйдите и войдите снова

### "No servers available"
**Причина:** Нет серверов в БД
**Решение:** Добавьте сервер через админку

### Не перенаправляет на login
**Причина:** Старый токен в localStorage
**Решение:** Очистите localStorage в DevTools
