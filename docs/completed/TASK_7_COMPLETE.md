## Как использовать

### Через UI (рекомендуется)

1. Откройте админ-панель: http://localhost:8080/admin
2. Войдите как `admin@3wg.ru` / `admin123`
3. Перейдите на вкладку "Серверы"
4. Нажмите "Подробнее" на сервере с WGDashboard
5. Выберите конфигурацию (wg0, wg1, и т.д.)
6. Нажмите кнопку "Добавить пир"
7. Заполните форму:
   - **Имя**: любое имя (например, `my-laptop`)
   - **Публичный ключ**: валидный WireGuard ключ (44 символа base64)
   - **Разрешенные IP**: IP адрес в формате CIDR (например, `10.0.0.100/32`)
8. Нажмите "Добавить"

### Генерация ключей

```bash
# Linux/Mac/Windows с WireGuard
wg genkey | wg pubkey

# Пример вывода:
8Jx9vkKt5Qy7wN2mP3rL4sT6uV8xZ0aB1cD2eF3gH4i=
```

### Через API

```bash
POST /api/v1/admin/servers/:id/wg/peers
Authorization: Bearer <token>
Content-Type: application/json

{
  "name": "my-device",
  "public_key": "8Jx9vkKt5Qy7wN2mP3rL4sT6uV8xZ0aB1cD2eF3gH4i=",
  "allowed_ips": ["10.0.0.100/32"],
  "config": "wg0"
}
```

## Важные замечания

### ⚠️ Валидация ключей
WireGuard строго проверяет формат публичных ключей:
- Должен быть base64
- Длина 44 символа
- Заканчивается на `=`

Невалидные ключи будут отклонены с ошибкой.

### 📝 Формат IP адресов
- Используйте CIDR нотацию: `10.0.0.100/32`
- Можно указать несколько через запятую: `10.0.0.100/32, 10.0.1.100/32`
- Убедитесь что IP не конфликтует с существующими пирами

### 🔄 Автоматическое обновление
После добавления пира:
- Список пиров обновляется автоматически
- Новый пир появляется в списке
- Конфигурация WireGuard обновляется на сервере
