## Как это работает

### Подсчет пиров

1. **Источник данных**: Таблица `wg_peers_cache` в PostgreSQL
2. **Запрос**: `SELECT COUNT(*) FROM wg_peers_cache WHERE server_id = ?`
3. **Кэширование**: Данные берутся из кэша, который обновляется каждые 30 секунд
4. **Производительность**: Быстрый подсчет без обращения к WGDashboard

### Отображение в UI

```
┌─────────────────────────────────────┐
│ Пиры:                    15 / 100   │ ← Текущие / Максимум
│ ████████░░░░░░░░░░░░░░░░░░░░░░░░░░ │ ← Прогресс-бар (15%)
└─────────────────────────────────────┘
```

**При загрузке >80%:**
```
┌─────────────────────────────────────┐

│ ████████████████████████████░░░░░░░ │ ← Красный градиент
│ ⚠️ Сервер близок к лимиту!          │ ← Предупреждение
└─────────────────────────────────────┘
```

## Цветовая схема

| Загрузка | Цвет | Градиент | Значение |
|----------|------|----------|----------|
| 0-60% | 🟢 Зеленый | `from-green-600 to-green-400` | Нормальная загрузка |
| 60-80% | 🟡 Желтый | `from-yellow-600 to-yellow-400` | Средняя загрузка |
| 80-100% | 🔴 Красный | `from-red-600 to-red-400` | Высокая загрузка |

## Интеграция с балансировщиком

Эта реализация является основой для будущей автоматической балансировки нагрузки:

1. **Текущее состояние**: Отображение загрузки серверов в админке
2. **Следующий шаг**: Интеграция с `backend/services/load_balancer.go`
3. **Будущее**: Автоматическое распределение пиров при создании VPN ключей

## Пример использования

### В админке

1. Открыть админку → вкладка "Серверы"
2. Каждая карточка сервера показывает:
   - Текущее количество пиров
   - Максимальное количество (из поля `max_users`)
   - Процент загрузки (визуально)
   - Предупреждение при высокой загрузке

### API эндпоинты

**Получить список серверов:**
```bash
GET /api/v1/servers
```

**Ответ:**
```json
{
  "servers": [
    {
      "id": 1,
      "name": "Amsterdam Server",
      "max_users": 100,
      "current_peers": 15,
      "status": "active",
      ...
    }
  ],
  "total": 1
}
```

**Получить конкретный сервер:**
```bash
GET /api/v1/servers/1
GET /api/v1/admin/servers/1
```

**Ответ:**
```json
{
  "id": 1,
  "name": "Amsterdam Server",
  "max_users": 100,
  "current_peers": 15,
  "status": "active",
  ...
}
```
