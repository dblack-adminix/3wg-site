## Что исправлено

### Проблема 1: Не показывало текущее количество пиров

**Причина**: Ошибка в индексе таблицы `wg_peers_cache`
- Был создан индекс только на `public_key`
- Нужен был составной индекс на `(server_id, config_name, public_key)`
- Из-за этого sync worker не мог сохранять пиры в кэш

**Решение**:
1. Исправлен `backend/models/cache.go` - добавлен правильный составной индекс
2. Создана миграция `backend/migrations/fix_peers_cache_index.sql`
3. Применена миграция к БД
4. Перезапущен бэкенд

**Результат**: Пиры успешно синхронизируются каждые 30 секунд

### Проблема 2: Нет поля для редактирования max_users

**Решение**:
1. Добавлено поле `max_users` в `formData` (по умолчанию 100)
2. Добавлено поле "Максимум пиров" в форму создания/редактирования
3. Добавлена валидация (min=1, max=10000)
4. Обновлена логика сохранения

**Результат**: Теперь можно задавать лимит пиров при создании и редактировании сервера

## Пример использования

### Создание сервера

1. Открыть админку → вкладка "Серверы"
2. Нажать "Добавить сервер"
3. Заполнить форму:
   - Название сервера: "Amsterdam Server"
   - URL WGDashboard: "http://46.30.43.35:10086"
   - API ключ: "ваш_ключ"
   - **Максимум пиров: 100** ← НОВОЕ ПОЛЕ
4. Нажать "Создать"
