# Тестирование генерации VPN ключей

## Статус: ✅ Готово к тестированию

## Что протестировать

### 1. Страница Generator (UI)

**URL:** http://localhost:5173/generator

**Шаги:**
1. Откройте страницу Generator
2. Выберите протокол (WireGuard или AmneziaWG)
   - Должна меняться цветовая схема (красный/оранжевый)
3. Выберите сервер из списка
   - Должны отображаться реальные серверы из БД
   - Флаг страны, название, локация
   - Загрузка (пиры/макс)
   - Симулированная задержка
4. Введите имя ключа (или оставьте автогенерированное)
5. Нажмите "GENERATE CONFIG"
6. Наблюдайте анимацию терминала с прогрессом
7. После завершения должен появиться конфиг
8. Попробуйте:
   - Скопировать конфиг (кнопка "Copy to Clipboard")
   - Скачать .conf файл (кнопка "Download .conf")

**Ожидаемый результат:**
- Конфиг создается успешно
- IP адрес уникальный (10.0.0.X)
- Для AmneziaWG добавлены дополнительные параметры
- Файл скачивается с правильным именем

### 2. Страница Keys (Управление ключами)

**URL:** http://localhost:5173/keys

**Шаги:**
1. Откройте страницу Keys
2. Должны отображаться все созданные ключи
3. Раскройте карточку ключа
4. Попробуйте:
   - Скопировать конфиг
   - Скачать .conf файл
   - Показать QR код
   - Удалить ключ

**Ожидаемый результат:**
- Все ключи отображаются
- Действия работают корректно
- После удаления ключ исчезает из списка

### 3. API тестирование

**Создание ключа:**
```powershell
./test-create-key.ps1
```

**Создание нескольких ключей:**
```powershell
./test-multiple-keys.ps1
```

**Проверка уникальности IP:**
- Каждый новый ключ должен получать следующий IP
- 10.0.0.2, 10.0.0.3, 10.0.0.4, и т.д.

### 4. Проверка конфига

**WireGuard конфиг должен содержать:**
```
[Interface]
PrivateKey = <base64_key>
Address = 10.0.0.X/24
DNS = 1.1.1.1, 8.8.8.8

[Peer]
PublicKey = <base64_key>
Endpoint = <server_ip>:51820
AllowedIPs = 0.0.0.0/0, ::/0
PersistentKeepalive = 25
```

**AmneziaWG конфиг дополнительно:**
```
# AmneziaWG specific settings
Jc = 4
Jmin = 40
Jmax = 70
S1 = 86
S2 = 37
H1 = 1234567890
H2 = 9876543210
H3 = 5555555555
H4 = 1111111111
```

## Известные проблемы

1. **Mock ключи**: Ключи генерируются случайно, не через WireGuard tools
2. **Mock публичный ключ сервера**: Используется случайный ключ вместо реального
3. **Нет автодобавления на сервер**: Пир не добавляется автоматически в WGDashboard

## Следующие шаги

1. Установить WireGuard tools в Docker
2. Использовать `wg genkey` и `wg pubkey` для генерации ключей
3. Получать реальный публичный ключ сервера из WGDashboard
4. Автоматически добавлять пир на сервер при создании ключа
5. Автоматически удалять пир при удалении ключа
