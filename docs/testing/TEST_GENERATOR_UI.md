# Тестирование UI генератора ключей

## 🎯 Цель
Проверить что пользователь может создать VPN ключ через веб-интерфейс в личном кабинете.

## 📋 Шаги тестирования

### 1. Авторизация
1. Откройте http://localhost:8080/admin
2. Войдите как пользователь (не админ):
   - Email: `user@example.com` (или создайте нового)
   - Password: `password`

### 2. Переход на страницу Generator
1. После входа перейдите на http://localhost:8080/generator
2. Или нажмите на ссылку "Generator" в меню

### 3. Создание ключа

**Шаг 1: Выбор протокола**
- [ ] Выберите WireGuard (красная подсветка)
- [ ] Или выберите AmneziaWG (оранжевая подсветка)

**Шаг 2: Выбор сервера**
- [ ] Должен отобразиться список серверов из БД
- [ ] Проверьте что показываются:
  - Флаг страны
  - Название сервера
  - Локация
  - Загрузка (X/100 пиров)
  - Симулированная задержка (~XXms)
  - Статус (зеленая точка = активен)
- [ ] Выберите сервер (например, "Amsterdam Test Server")

**Шаг 3: Имя ключа**
- [ ] Поле должно автоматически заполниться (например, "Amsterdam_Test_Server_wireguard_20260131")
- [ ] Можете изменить имя на свое

**Шаг 4: Генерация**
- [ ] Нажмите кнопку "GENERATE CONFIG"
- [ ] Должна появиться анимация терминала с прогрессом
- [ ] Прогресс-бар должен заполняться
- [ ] Через ~3 секунды должен появиться конфиг

**Шаг 5: Действия с конфигом**
- [ ] Нажмите "Copy to Clipboard" - должно появиться уведомление
- [ ] Вставьте из буфера обмена - конфиг должен быть скопирован
- [ ] Нажмите "Download .conf" - файл должен скачаться
- [ ] Откройте скачанный файл - проверьте содержимое

### 4. Проверка в личном кабинете

1. Перейдите на http://localhost:8080/keys
2. Проверьте что созданный ключ отображается в списке:
   - [ ] Имя ключа
   - [ ] IP адрес (10.16.11.X)
   - [ ] Протокол (WireGuard или AmneziaWG)
   - [ ] Статус (active)
   - [ ] Дата создания

3. Раскройте карточку ключа:
   - [ ] Кнопка "COPY" - копирует конфиг
   - [ ] Кнопка "QR" - показывает QR код
   - [ ] Кнопка ".CONF" - скачивает файл
   - [ ] Кнопка "REGENERATE" - регенерирует ключ (в разработке)
   - [ ] Кнопка удаления (красная) - удаляет ключ

### 5. Проверка на сервере (опционально)

Если есть доступ к админке:
1. Откройте http://localhost:8080/admin
2. Перейдите на вкладку "Servers"
3. Найдите сервер на котором создали ключ
4. Раскройте детали сервера
5. Проверьте что пир появился в списке:
   - [ ] Имя пира совпадает с именем ключа
   - [ ] IP адрес совпадает
   - [ ] Статус "stopped" (пока не подключен)

## 🐛 Возможные проблемы

### Проблема 1: "No servers available"
**Причина:** Нет серверов в БД или они не активны
**Решение:** Добавьте сервер через админку

### Проблема 2: "Failed to create key"
**Причина:** Ошибка на backend
**Решение:** Проверьте логи backend: `docker logs vpn_backend`

### Проблема 3: Ключ создался но пир не появился на сервере
**Причина:** WGDashboard не настроен или недоступен
**Решение:** Проверьте настройки сервера (wg_dashboard_url, wg_dashboard_key)

### Проблема 4: IP адрес неправильный (10.0.0.X вместо 10.16.11.X)
**Причина:** Старая версия backend
**Решение:** Пересоберите backend: `docker-compose up -d --build`
