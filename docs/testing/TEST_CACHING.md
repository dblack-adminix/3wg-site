# Тестирование оптимизации кэширования

## Что проверить

### 1. Список серверов (главная страница админки)
1. Откройте админ-панель: http://localhost:5173/admin
2. Перейдите на вкладку "Серверы"
3. **Ожидаемое поведение**:
   - ✅ Список серверов появляется **мгновенно** (< 1 сек)
   - ✅ Метрики (CPU, RAM, Disk) показывают скелетон-анимацию
   - ✅ Через 1-2 секунды метрики подгружаются из кэша
   - ✅ Нет блокирующего спиннера

### 2. Детали сервера
1. Нажмите "Подробнее" на любом сервере
2. **Ожидаемое поведение**:
   - ✅ Базовая информация (название, IP) появляется **мгновенно**
   - ✅ Конфигурации и пиры показывают скелетон
   - ✅ Через 1-2 секунды данные подгружаются из кэша
   - ✅ Нет блокирующего спиннера

### 3. Переключение между конфигурациями
1. Если у сервера несколько конфигураций (wg0, wg1), переключайтесь между ними
2. **Ожидаемое поведение**:
   - ✅ Переключение **мгновенное** (данные из кэша)
   - ✅ Нет задержек и спиннеров

### 4. Автообновление
1. Оставьте страницу открытой на 30 секунд
2. **Ожидаемое поведение**:
   - ✅ Метрики обновляются каждые 2 секунды (незаметно)
   - ✅ Данные обновляются каждые 15-30 секунд (незаметно)
   - ✅ Нет спиннеров при автообновлении

### 5. Первый запуск (пустой кэш)
1. Остановите бэкенд: `docker-compose stop backend`
2. Очистите кэш: `docker exec -i vpn_postgres psql -U postgres -d vpn_3wg -c "TRUNCATE wg_peers_cache, wg_configs_cache, system_status_cache;"`
3. Запустите бэкенд: `docker-compose start backend`
4. Откройте админ-панель
5. **Ожидаемое поведение**:
   - ✅ Страница открывается мгновенно
   - ✅ Данные подгружаются из WGDashboard (fallback)
   - ✅ Через 2-30 секунд кэш заполняется
   - ✅ Последующие загрузки мгновенные

## Логи для проверки

### Backend логи
```bash
cd backend
docker-compose logs -f backend | Select-String -Pattern "Worker|Synced"
```

Должны видеть:
```
[Metrics Worker] Starting initial sync...
[Data Worker] Starting initial sync...
[Metrics] Synced server 1 (CPU: 1.0%, RAM: 41.5%, Disk: 11.4%)
[Data] Synced server 1 config wg0 (14 peers)
```

### Browser Console
Откройте DevTools (F12) → Console

Должны видеть:
```
[Cache] Config cache is empty, loading from WGDashboard...
```
(только при первом запуске с пустым кэшом)

## Проблемы?

### Спиннер все еще крутит
- Проверьте что бэкенд запущен: `docker-compose ps`
- Проверьте логи: `docker-compose logs backend`
- Убедитесь что sync workers работают

### Данные не обновляются
- Проверьте что sync workers запущены (см. логи выше)
- Проверьте что нет ошибок в логах
- Перезапустите бэкенд: `docker-compose restart backend`

### Ошибки в консоли браузера
- Проверьте что бэкенд доступен: http://localhost:3000/api/v1/servers
- Проверьте CORS настройки
- Проверьте что токен авторизации валидный
