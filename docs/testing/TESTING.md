## Тестирование авторизации

### Обычный пользователь

1. **Регистрация нового пользователя:**
   - Нажмите "Личный кабинет" в шапке
   - Вы попадете на страницу `/login`
   - Нажмите "Зарегистрироваться"
   - Введите email и пароль (минимум 6 символов)
   - Нажмите "Зарегистрироваться"
   - Вы автоматически войдете и попадете в личный кабинет

2. **Вход существующего пользователя:**
   - Нажмите "Личный кабинет"
   - Введите email и пароль
   - Нажмите "Войти"
   - Вы попадете в личный кабинет

### Администратор

1. **Вход в админку:**
   - Перейдите на http://localhost:8080/admin
   - Или нажмите "Вход для администратора" на странице логина
   - Введите:
     - Email: `admin@3wg.ru`
     - Password: `admin123`
   - Нажмите "Войти"
   - Вы попадете в админ-панель

2. **Проверка админки:**
   - Должна показаться реальная статистика из бэкенда
   - Всего пользователей: 1 (или больше если регистрировали)
   - Активные серверы: 0
   - Доход за месяц: ₽0

## Проверка API интеграции

### Через браузер (DevTools)

1. Откройте DevTools (F12)
2. Перейдите на вкладку Network
3. Войдите в админку
4. Вы должны увидеть запросы:
   - `POST /api/v1/auth/login` - логин
   - `GET /api/v1/users/me` - получение профиля
   - `GET /api/v1/admin/dashboard` - статистика админки

### Через PowerShell

```powershell
# Логин
$body = @{email="admin@3wg.ru";password="admin123"} | ConvertTo-Json
$response = Invoke-WebRequest -Uri "http://localhost:3000/api/v1/auth/login" -Method POST -ContentType "application/json" -Body $body -UseBasicParsing
$data = $response.Content | ConvertFrom-Json
$token = $data.token

# Получить профиль
$headers = @{Authorization="Bearer $token"}
Invoke-WebRequest -Uri "http://localhost:3000/api/v1/users/me" -Headers $headers -UseBasicParsing | Select-Object -ExpandProperty Content

# Админ дашборд
Invoke-WebRequest -Uri "http://localhost:3000/api/v1/admin/dashboard" -Headers $headers -UseBasicParsing | Select-Object -ExpandProperty Content
```

## Проверка JWT токенов

1. Войдите в систему
2. Откройте DevTools → Application → Local Storage
3. Должен быть ключ `auth_token` с JWT токеном
4. Выйдите из системы - токен должен удалиться

## Известные проблемы

### Если не работает авторизация:
1. Проверьте что бэкенд запущен: `.\manage.ps1 status`
2. Проверьте логи бэкенда: `.\manage.ps1 logs`
3. Проверьте что админ создан: `.\manage.ps1 create-admin`

### Если показывает "Unauthorized":
1. Очистите localStorage в браузере
2. Перезагрузите страницу
3. Попробуйте войти заново

### Если CORS ошибки:
1. Проверьте что фронтенд на порту 8080
2. Проверьте что бэкенд на порту 3000
3. В бэкенде настроен CORS для localhost:8080
